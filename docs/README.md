# Documentation

This directory is reserved for durable maintainer-facing documentation.

Recommended placement:

- `docs/releases/` for release notes and release preparation records.
- `docs/experiments/` for experiment write-ups that correspond to `experiments/` branches.
- `docs/design/` for longer-lived design notes and architecture decisions.

Short user-facing setup instructions should stay in the top-level README files. Detailed operational notes belong here.
