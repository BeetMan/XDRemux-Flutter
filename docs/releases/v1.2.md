# XDRemux v1.2

Prepared from the current main branch after v1.1.

## Highlights

- Keeps the Swift CLI as the recommended macOS conversion entry point.
- Documents the current hybrid input-processing branch as the production default.
- Preserves the Python CLI as the cross-platform entry point.
- Clarifies OPPO compatibility behavior: clean Apple/ImageIO-compatible output by default, with `--oppo-compat` reserved for OPPO Gallery recognition signals and private compatibility data.
- Standardizes repository layout documentation and moves reusable local automation into `scripts/`.

## User impact

Users should continue invoking the main converters through `xdremux/swift/XDRemux.swift` or `xdremux/python/XDRemux.py`.

The macOS app helper script moved from `script/build_and_run.sh` to `scripts/build_and_run.sh`.

## Validation status

This release-preparation pass inspected the repository metadata, README entry points, current release state, and the checked-in helper script. A full macOS ImageIO/Xcode validation run was not performed in this Linux container because the Swift CLI imports Apple-only frameworks.
